# FLOOD RTL v4 独立验证报告 v1

## 目的

这份报告使用没有参与 v4 公式拟合的新 RTL 样本，检查候选 v4 校准公式是否具备外推能力。

注意：这里的“独立”是指这些点没有用于生成 v4 修正项；它们仍然是小规模 RTL testbench 样本，不等价于完整 workload RTL 仿真。

## 样本设计

本次新增 6 个点：

- `k=3/group_size=8/cout=6`：位于已拟合的 `cout=4` 和 `cout=8` 之间。
- `k=3/group_size=8/cout=12`：超出已拟合的 `cout=8`，检查外推。
- `k=3/group_size=8/cout=6/cin=2`：检查多 Cin 的 `final-3` 假设。
- `k=1/group_size=16/cout=10`：位于已拟合曲线中间。
- `k=1/group_size=16/cout=16`：超出已拟合的 `cout=12`，检查外推。
- `k=1/group_size=16/cout=12/res_cols=2`：检查高 group 下空间重复执行。

## 结果

| 指标 | v3 | v4 |
|---|---:|---:|
| 有效样本数 | 5 | 5 |
| 完全命中数 | 0 | 5 |
| 平均绝对误差 | 22.4432% | 0.0% |
| 最大绝对误差 | 26.5199% | 0.0% |

## 关键结论

- v4 在 5 个有效独立 RTL 点上全部命中，说明 `k=3/group_size=8` 交互项和 `k=1/group_size=16` 曲线修正具有较强可信度。
- v3 在同一批点上平均误差超过 22%，进一步证明原公式不能作为论文级性能模型。
- `u06_k1_c12_g16_rc2` 出现 `246;0`，说明 `group_size=16` 下不只是多 Cin，空间重复执行也会触发后续 run 为 0 的控制异常。
- 因此，目前 v4 可以作为“校准后的 RTL-aware 小规模性能模型”，但高 group 重复执行路径必须先修 testbench，再补完整验证。

3. 选 2-3 个真实 workload layer，使用 v4 预测并运行对应 RTL 小层验证，形成论文表格。
