### 7. group16 空间重复 X 根因定位

```text

```

内容：

- 定位 `group_size=16, res_cols=2` 空间重复第二列输出 X 的来源。
- 探针确认 feature、weight、Cluster 输出均无 X。
- X 来自 `OutRouterPlanePost` 读取未初始化 output SRAM 高地址 `513..523`。
- 使用 SRAM memory 初始清零后，同一 case 的 `x_count` 变为 0，周期保持 `246;56`。

## 当前结论

1. FLOOD RTL 已经可以跑出周期。
2. 已修复多 Cin 重复执行的 0 周期问题，并定位 group16 空间重复 X 主要来自 output SRAM 未初始化读。
3. 目前有 22 个完整 RTL bring-up case。
4. 已经生成 workload 级 RTL-calibrated projection。
5. 下一步应使用 Verilator 或更快仿真器跑代表性 workload layer，形成论文级 RTL validation 表。
